# Unsaved Changes Banner — Data Contract

## Summary

When a user edits or deletes a term in the Data Contract view, a yellow warning banner appears below the toolbar listing the pending changes. A "✕ Cancel all" link on the right lets users discard all changes after confirmation.

## Trigger Condition

The banner is shown when:
- The user is viewing the **latest** contract version (`is_latest = True`)
- `_pending_edit_count(pending) > 0`

It disappears automatically when the pending state is cleared (after save or cancel all).

## Banner Content

**Layout:** Single row. Left side: change summary text. Right side: "✕ Cancel all" link (red).

**Change summary format:**
- Singular: `1 unsaved change: deleted order_status`
- Plural (≤3 items): `3 unsaved changes: deleted order_status, edited region_code.classification, deleted legacy_id`
- Plural (>3 items): `4 unsaved changes: deleted order_status, edited region_code.classification, deleted legacy_id and 1 more`

**Per-change label format:**
| Change type | Label |
|---|---|
| Term deletion | `deleted {col}` |
| Governance field edit | `edited {col}.{field}` |
| Test rule edit | `edited {col} rule` |

Labels are generated by a new `_format_pending_labels(pending)` helper that iterates the `dc_pending` dict and returns a list of human-readable strings.

## Cancel All Flow

1. User clicks "✕ Cancel all" in the banner.
2. A `@st.dialog` confirmation opens: *"Discard {N} unsaved change(s)? This cannot be undone."* with **Discard** and **Go back** buttons.
3. On **Discard**: clear `dc_pending:{table_group_id}` and reload the original YAML from DB into `dc_yaml:{table_group_id}`, then `st.rerun()`.
4. On **Go back**: close dialog, no state change.

Reloading from DB (rather than just clearing pending) ensures the in-memory YAML is fully restored to its last-saved state, not left in a partially-mutated state.

## Placement in `data_contract.py`

The banner slots in after the toolbar block and before the existing staleness banner:

```
toolbar (version picker + Regenerate + Save)
← NEW: unsaved changes banner (if is_latest and pending > 0)
staleness banner (if is_latest and stale)
review changes panel (on user request)
```
